````md id="zc9m2r"
# Fix Mobile Responsive Design Issues Across the Entire System

The mobile responsiveness of the system is currently broken in multiple sections.

The UI is overflowing, elements are getting cut off, tables are not adapting properly, spacing is inconsistent, and the layout is not optimized for smaller screens.

Screenshots attached for reference.

This needs a COMPLETE responsive overhaul for mobile and tablet devices.

---

# CRITICAL ISSUES OBSERVED

## Services Page Issues

### Problems

- Table overflows horizontally
- Search area has excessive empty space
- Table columns are compressed and unreadable
- Company/service tags overflow badly
- Pagination alignment is broken
- Floating action button overlaps content
- Layout hierarchy is poor on mobile
- Card spacing is inconsistent

---

## Companies Page Issues

### Problems

- Table width exceeds viewport
- Linked services badges overflow
- Columns become unreadable
- Empty white spaces in filter/search area
- Table structure is not mobile optimized
- Pagination/footer alignment issues
- Content density is too high for small screens

# TABLE RESPONSIVENESS FIX

## REQUIRED

Desktop-style wide tables SHOULD NOT render directly on mobile.

Instead:

### Mobile Behaviour

Convert tables into:

- stacked cards
OR
- responsive scroll containers

depending on the data type.

---

# OPTION 1 (Preferred)

## Convert rows into mobile cards

Example:

```txt
Service Name
CRM Implementation

Category
Software

Linked Companies
TechNova Solutions
FinEdge Partners
```

This is MUCH cleaner on mobile.

---

# OPTION 2

If keeping tables:

```css
overflow-x: auto;
-webkit-overflow-scrolling: touch;
```

BUT:

- maintain proper padding
- prevent text clipping
- prevent badge overlap
- ensure readable spacing

---

# SEARCH + FILTER SECTION FIX

Current layout wastes vertical space.

## REQUIRED

On mobile:

- stack filters vertically
- full width inputs
- proper spacing between controls
- remove unnecessary empty containers

Example:

```txt
[ Search ]

[ Filter 1 ]

[ Filter 2 ]
```

NOT:

```txt
tiny compressed horizontal layout
```

---

# BADGE/TAG FIXES

## Linked Services / Companies badges

Currently:

- overflowing
- wrapping badly
- unreadable

## REQUIRED

Use:

```css
flex-wrap: wrap;
gap: 6px;
max-width: 100%;
```

Badges should:

- wrap cleanly
- maintain spacing
- never overflow outside container

---

# FLOATING ACTION BUTTON FIX

Current FAB overlaps content.

## REQUIRED

On mobile:

```css
bottom: 16px;
right: 16px;
z-index: proper;
```

Also:

- ensure pagination/content is not hidden behind FAB
- add bottom padding to scroll area

---

Current pagination alignment is broken.

## REQUIRED

On mobile:

- stack pagination properly
- reduce spacing
- center align controls
- avoid overflow

Example:

< 1 >
```
````
