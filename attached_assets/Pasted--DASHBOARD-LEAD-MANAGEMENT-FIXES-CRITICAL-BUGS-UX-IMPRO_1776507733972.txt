# 🐞 ISSUE 1: DEAL VALUE ROUNDING ERROR

---

## ❗ PROBLEM

When entering:

```plaintext
₹10,00,000
```

System displays:

```plaintext
₹9,99,997
```

---

## 🎯 ROOT CAUSE

* Floating point precision issue (likely JS number handling)
* Improper rounding or conversion

---

## ✅ FIX

### ALWAYS STORE VALUES AS INTEGERS

```js
// Store in paise or smallest unit OR as integer rupees
value: Number(inputValue)
```

---

### SAFE FORMAT FUNCTION

```js
function formatCurrency(value) {
  return new Intl.NumberFormat('en-IN', {
    maximumFractionDigits: 0
  }).format(Math.round(value));
}
```

---

### ❗ ENSURE:

* No floating calculations like `value * 1000.0`
* Use `Math.round()` before storing/displaying

# 📊 ISSUE 3: CLOSURE FILTER (WON vs LOST)

```plaintext

```

But user cannot differentiate:

* Won ✅
* Lost ❌

---

## ✅ SOLUTION

---

### STEP 1: ENSURE STATUS FIELD EXISTS

```js

```

---

### STEP 2: UI IMPROVEMENT

In table:

```plaintext

Closed – Lost  (Red)
Closed – Postponed (Yellow)
Closed – Ghost (Gray)
```

---

### STEP 3: FILTER UPDATE

Add sub-filter:

```plaintext

[ All | Won | Lost | Postponed | Ghost ]
```

### STEP 4: CHART COLORS
