# 🔧 Spark Lead Hub — Critical UI Fixes: Sheet, Scrolling, Dropdowns, Chart & Pipeline

Deep audit completed across all 8 screenshots. Every issue is catalogued below with an exact fix. No issue is left unaddressed.

## 🔴 COMPLETE AUDIT — Every Issue Found

### Lead Detail Sheet
1. **Two close (×) buttons** — one in the sheet header, one floating top-right. Remove the duplicate. Keep only ONE, positioned consistently top-right inside the sheet header.
2. **Scroll bleeds through** — scrolling inside the sheet scrolls the page behind it. The sheet body must have `overflow-y: auto` and the backdrop must block scroll propagation on the underlying page.
3. **Lead header is underwhelming** — stage badge, status badge, lead type badge, and deal value are tiny, flat, and unclear. They need visual weight — proper colored badges, clear hierarchy, readable sizes.
4. **Stage shows "Select stage…" on existing lead** — the stage/status are not being populated from the lead's existing `pipeline_stage_id` / `pipeline_status_id`. This is a data-loading bug.
5. **Details/Notes tabs are broken visually** — they look like plain unstyled text, no active state, no tab indicator. Must match the system tab design exactly.
6. **Spacing/padding is completely off** — fields are either too cramped or too spaced, inconsistent throughout the sheet.
7. **Pipeline progress bar takes up too much vertical space** and the stage labels are tiny/overlapping.

### Dropdowns
8. **Stage dropdown opens with bright blue highlight** — same blue-highlight bug. Must use teal system highlight.
9. **All dropdowns in the sheet (Stage, Status, Lead Type, Service, Companies, Lead Owner, Deal Handler) are unstyled** — they look like default browser selects. Must match the dashboard filter dropdown style exactly (dark background, border, teal active highlight, proper animation).
10. **"Lead Type" dropdown in sheet shows raw select** — missing the emoji prefix on the selected value display.

### Pipeline Chart (Dashboard)
11. **X-axis labels show raw snake_case** — `lead_initiation`, `qualification_analysis`, `proposal_negotiation`, `closure`. Must show display names: `Lead Initiation`, `Qualification & Analysis`, `Proposal & Negotiation`, `Closure`.
12. **One stage missing from chart** — `qualification_analysis` bar appears but stage `qualification_analysis` shows 0 in some renders because the chart is using the `name` column instead of joining on `id` to get `display_name`. Fix the data query.
13. **All bars are the same gray color** — bars must use per-stage colors from `pipeline_stages.color`.
14. **Tooltip text is black on dark background** — completely invisible. Fix tooltip contentStyle to use light text on dark panel.
15. **Tooltip label shows raw snake_case** — `lead_initiation : 2 Leads`. Must show `Lead Initiation — 2 Leads`.

### Notes Tab
16. **Notes tab send button is a teal floating pill** on the right edge — looks detached and broken. Must be a proper button inside the input area.
17. **"No notes yet" text is centered but has no icon** — needs an empty state with icon.

## PART 1 — Fix Scroll Bleed (Body Scroll Lock)

When the lead detail sheet is open, the page behind it must NOT scroll. Fix both CSS and JS:

```tsx
// In LeadDetailSheet.tsx — when sheet opens/closes:
useEffect(() => {
  if (isOpen) {
    // Lock body scroll
    const scrollY = window.scrollY;
    document.body.style.position = 'fixed';
    document.body.style.top = `-${scrollY}px`;
    document.body.style.width = '100%';
    document.body.style.overflow = 'hidden';
  } else {
    // Restore body scroll
    const scrollY = document.body.style.top;
    document.body.style.position = '';
    document.body.style.top = '';
    document.body.style.width = '';
    document.body.style.overflow = '';
    window.scrollTo(0, parseInt(scrollY || '0') * -1);
  }
  return () => {
    // Cleanup on unmount
    document.body.style.position = '';
    document.body.style.top = '';
    document.body.style.width = '';
    document.body.style.overflow = '';
  };
}, [isOpen]);
```

## PART 5 — Fix Pipeline Progress Bar (Compact, Non-Overlapping)

The current progress bar is too tall and the labels overlap on smaller widths. Rebuild it compact:

```tsx
const PipelineProgressBar = ({ stages, currentStageId, currentStatus }) => {
  const currentIndex = stages.findIndex(s => s.id === currentStageId);

  return (
    <div className="pipeline-bar" role="progressbar">
      {stages.map((stage, i) => {
        const isPast    = i < currentIndex;
        const isCurrent = stage.id === currentStageId;
        const isFuture  = i > currentIndex;
        const isLast    = i === stages.length - 1;

        return (
          <React.Fragment key={stage.id}>
            <div className={`pb-step ${isPast ? 'past' : ''} ${isCurrent ? 'current' : ''} ${isFuture ? 'future' : ''}`}>
              {/* Track line before dot (except first) */}
              {i > 0 && (
                <div
                  className={`pb-line ${isPast || isCurrent ? 'filled' : ''}`}
                  style={isPast || isCurrent ? { background: stages[i - 1].color } : {}}
                />
              )}
              {/* Dot */}
              <div
                className="pb-dot"
                style={isCurrent || isPast ? { background: stage.color, borderColor: stage.color } : {}}
              >
                {isPast && <Check size={8} strokeWidth={3} />}
              </div>
              {/* Label */}
              <div className="pb-label">
                <span className="pb-stage-name">{stage.displayName}</span>
                {isCurrent && currentStatus && (
                  <span className="pb-status-name" style={{ color: stage.color }}>{currentStatus}</span>
                )}
              </div>
            </div>
            {/* Connector line between steps */}
            {!isLast && (
              <div
                className={`pb-connector ${isPast ? 'filled' : ''}`}
                style={isPast ? { background: stage.color } : {}}
              />
            )}
          </React.Fragment>
        );
      })}
    </div>
  );
};
```

## PART 7 — Rebuild ALL Dropdowns to Match Dashboard Filter Style

The dashboard "All Types" filter dropdown is the reference design. ALL dropdowns across the entire system must look and behave identically.

### Reference: Dashboard Filter Dropdown (what works)
- Dark background panel
- Teal border on trigger when open
- Teal-tinted background on selected/hovered items
- Emoji prefixes on lead type options
- Smooth dropdown-in animation

### Universal Custom Select Component

Replace ALL native `<select>` elements in the lead sheet with this custom component:

```tsx
// components/CustomSelect.tsx
interface Option {
  value: string;
  label: string;
  prefix?: React.ReactNode;   // emoji or icon
  color?: string;             // dot color
  disabled?: boolean;
}

interface Props {
  value: string | null;
  options: Option[];
  placeholder?: string;
  onChange: (value: string) => void;
  disabled?: boolean;
  className?: string;
}

export const CustomSelect = ({ value, options, placeholder = 'Select…', onChange, disabled, className }: Props) => {
  const [open, setOpen] = useState(false);
  const ref = useRef<HTMLDivElement>(null);
  const selected = options.find(o => o.value === value);

  // Close on outside click
  useEffect(() => {
    const handler = (e: MouseEvent) => {
      if (ref.current && !ref.current.contains(e.target as Node)) setOpen(false);
    };
    document.addEventListener('mousedown', handler);
    return () => document.removeEventListener('mousedown', handler);
  }, []);

  // Close on Escape
  useEffect(() => {
    const handler = (e: KeyboardEvent) => { if (e.key === 'Escape') setOpen(false); };
    document.addEventListener('keydown', handler);
    return () => document.removeEventListener('keydown', handler);
  }, []);

  return (
    <div ref={ref} className={`cselect ${className ?? ''} ${disabled ? 'disabled' : ''}`}>
      {/* Trigger */}
      <div
        className={`cselect-trigger ${open ? 'open' : ''} ${!value ? 'placeholder' : ''}`}
        onClick={() => !disabled && setOpen(o => !o)}
        role="combobox"
        tabIndex={disabled ? -1 : 0}
        onKeyDown={e => { if (e.key === 'Enter' || e.key === ' ') { e.preventDefault(); !disabled && setOpen(o => !o); } }}
      >
        <span className="cselect-value">
          {selected ? (
            <>
              {selected.color && <span className="cselect-dot" style={{ background: selected.color }} />}
              {selected.prefix && <span className="cselect-prefix">{selected.prefix}</span>}
              {selected.label}
            </>
          ) : (
            <span className="cselect-placeholder">{placeholder}</span>
          )}
        </span>
        <ChevronDown
          size={14}
          className="cselect-chevron"
          style={{ transform: open ? 'rotate(180deg)' : 'none', transition: 'transform 150ms ease' }}
        />
      </div>

      {/* Dropdown panel */}
      {open && (
        <>
          <div className="cselect-backdrop" onClick={() => setOpen(false)} />
          <div className="cselect-panel">
            {options.map(option => (
              <div
                key={option.value}
                className={`cselect-option ${value === option.value ? 'is-selected' : ''} ${option.disabled ? 'is-disabled' : ''}`}
                onClick={() => {
                  if (!option.disabled) {
                    onChange(option.value);
                    setOpen(false);
                  }
                }}
                role="option"
                aria-selected={value === option.value}
              >
                {option.color && <span className="cselect-dot" style={{ background: option.color }} />}
                {option.prefix && <span className="cselect-prefix">{option.prefix}</span>}
                <span className="cselect-option-label">{option.label}</span>
                {value === option.value && <Check size={13} className="cselect-check" />}
              </div>
            ))}
            {options.length === 0 && (
              <div className="cselect-empty">No options available</div>
            )}
          </div>
        </>
      )}
    </div>
  );
};
```
