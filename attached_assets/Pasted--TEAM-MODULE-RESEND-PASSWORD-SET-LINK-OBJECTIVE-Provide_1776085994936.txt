# 🔐 TEAM MODULE – RESEND PASSWORD SET LINK

---

# 🎯 OBJECTIVE

Provide Admins the ability to **resend password setup/reset link** to any user directly from the **Team Members table**.

---

# 📍 WHERE TO ADD

In the **Team → Members table**, under the **Actions column** (where delete icon exists):

👉 Add a new action:

* 🔁 Resend Password Link

---

# 🔧 BACKEND IMPLEMENTATION

---

## ✅ CREATE API

```http
POST /api/users/:userId/resend-password-link
```

---

## ✅ REQUEST

```json
{
  "userId": "123"
}
```

---

## ✅ BACKEND LOGIC

---

### 1. FETCH USER

```js
const user = await User.findById(userId);
```

---

### 2. GENERATE SECURE TOKEN

```js
const token = crypto.randomBytes(32).toString("hex");

const hashedToken = crypto
  .createHash("sha256")
  .update(token)
  .digest("hex");
```

---

### 3. SAVE TOKEN + EXPIRY

```js
user.passwordResetToken = hashedToken;
user.passwordResetExpires = Date.now() + 1000 * 60 * 30; // 30 mins

await user.save();
```

---

### 4. CREATE RESET LINK

```js
const resetLink = `${FRONTEND_URL}/set-password/${token}`;
```

---

### 5. SEND EMAIL

```js
await sendEmail({
  to: user.email,
  subject: "Set Your Password",
  html: `
    <p>Hello ${user.name},</p>
    <p>Click below to set your password:</p>
    <a href="${resetLink}">Set Password</a>
    <p>This link expires in 30 minutes.</p>
  `
});
```

---

# 🔒 SECURITY RULES (CRITICAL)

---

## ✅ DO NOT:

* Send password directly ❌
* Store raw token ❌

---

## ✅ MUST:

* Store only hashed token
* Expire link (30 mins)
* Single-use token

---

# 🔧 FRONTEND IMPLEMENTATION

---

## ✅ API CALL

```js
const resendPasswordLink = async (userId) => {
  try {
    await api.post(`/users/${userId}/resend-password-link`);
    showToast("Password reset link sent successfully");
  } catch (err) {
    showToast("Failed to send reset link");
  }
};
```

---

