# 🔔 FOLLOW-UP PAGE — “TODAY” & “OVERDUE” COUNTERS SHOULD FILTER TABLE DATA

---

# ❌ CURRENT ISSUE

On the Follow-Up page, the top summary counters:

* `0 today`
* `18 overdue`

are currently behaving like static indicators only.

When clicking these counters, the table data below does NOT get filtered.

This creates a UX inconsistency because users naturally expect these counters to behave as interactive filters.

---

# 🎯 REQUIRED BEHAVIOR

The summary counters should behave as:

* clickable filter chips/buttons
* real-time quick filters
* smart dashboard shortcuts

---

# ✅ REQUIRED FILTER LOGIC

---

# 1️⃣ CLICKING “TODAY”

When user clicks:

```text id="t2a4x9"
0 today
```

the table below should immediately filter and display ONLY:

* leads with follow-up date = today
* active/open leads only
* non-closed leads only

---

## FILTER CONDITIONS

```ts id="j7m0r3"
followUpDate === today
AND
leadStage !== "Closure"
```

OR

```ts id="7b4wz2"
status NOT IN [
  "Closed - Won",
  "Closed - Lost"
]
```

---

# 2️⃣ CLICKING “OVERDUE”

When user clicks:

```text id="0x9mrf"
18 overdue
```

the table below should immediately filter and display ONLY:

* overdue follow-ups
* follow-up date < today
* active/open leads only

---

## FILTER CONDITIONS

```ts id="u5a8zw"
followUpDate < today
AND
status NOT IN [
  "Closed - Won",
  "Closed - Lost"
]
```

---

# 3️⃣ ACTIVE FILTER STATE

* visually highlight the selected chip

* update table instantly
