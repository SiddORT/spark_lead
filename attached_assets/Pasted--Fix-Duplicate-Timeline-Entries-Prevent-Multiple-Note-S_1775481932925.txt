# 🎯 Objective

Fix two major issues:

### 1. Duplicate Timeline Entries

* Same event appearing multiple times
* Timeline looks incorrect and unreliable

---

### 2. Multiple Note Submissions

* No loading state on “Add Note”
* User clicks button multiple times
* Same note gets added multiple times

---

# ✅ EXPECTED BEHAVIOUR

---

## ✔ Timeline

* Each action should appear **ONLY ONCE**
* No duplicate entries
* Chronological and clean

---

## ✔ Notes

* On clicking **Add Note**:

  * Button should show loader
  * Button should be disabled
  * Only ONE note should be created

# 🧠 IMPLEMENTATION

---

## ✅ 1. Prevent Duplicate Notes (Frontend)

```tsx
const [isSubmitting, setIsSubmitting] = useState(false);

const handleAddNote = async () => {
  if (isSubmitting) return; // Prevent multiple clicks

  setIsSubmitting(true);

  try {
    await addNoteAPI(noteText);
    setNoteText("");
  } finally {
    setIsSubmitting(false);
  }
};
```

---

## ✅ 2. Add Loader + Disable Button

```tsx
<button
  onClick={handleAddNote}
  disabled={isSubmitting}
  className="btn-primary"
>
  {isSubmitting ? "Adding..." : "Add Note"}
</button>
```

---

## ✅ 3. Prevent Double Submit via Enter Key

```tsx
onKeyDown={(e) => {
  if (e.key === "Enter" && !e.shiftKey) {
    e.preventDefault();
    handleAddNote();
  }
}}
```
