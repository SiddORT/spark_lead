# 🚨 Spark Lead Hub — Critical Fixes: Auth, Email & Complete UI Polish

This prompt covers three categories of fixes that must ALL be completed before considering this done:
1. **Production login broken** — published app cannot authenticate
2. **Email delivery broken** — invites and access approvals never arrive
3. **UI uniformity & readability** — forms, dropdowns, typography, spacing all need a final pass

Do NOT skip any section. Test every fix end-to-end before marking complete.

---

## 🔴 PART 1 — Fix Production Login (Published App)

### Root Cause Analysis

The preview works but the published URL fails. This means one or more of these:

1. **Environment variables are not set in the Replit Secrets for production** — `.env` files are ignored in deployed/published Replit apps. Every variable must be in **Replit Secrets** (the lock icon in the sidebar).
2. **CORS is blocking requests** — the published domain is different from `localhost`, and the backend is rejecting cross-origin requests.
3. **API base URL is hardcoded to localhost** — the frontend is calling `http://localhost:3000/api/...` which works in preview but fails when published.
4. **Cookie/JWT not being sent cross-origin** — cookies set with `SameSite=Strict` or missing `Secure` flag won't work on the published HTTPS domain.
5. **Database connection string uses localhost** — if PostgreSQL URL points to `localhost`, it may not resolve correctly in the published environment.

---

### Fix 1.1 — Move ALL Environment Variables to Replit Secrets

Go to Replit → Secrets (lock icon) and add every single one of these. Do not rely on `.env` files in production:

```
DATABASE_URL        = postgresql://...  (full connection string)
JWT_SECRET          = (strong random string, min 32 chars)
JWT_EXPIRES_IN      = 7d
ADMIN_EMAIL         = admin@sparkleadhub.com
ADMIN_PASSWORD      = Admin@123!
PORT                = 3000
FRONTEND_URL        = https://YOUR-PUBLISHED-REPLIT-URL.repl.co
SMTP_HOST           = smtp.gmail.com
SMTP_PORT           = 587
SMTP_USER           = your@gmail.com
SMTP_PASS           = your-app-password
```

In code, access via `process.env.VARIABLE_NAME` — never hardcode any of these values.

**Verify every env var is loaded at server startup:**

### Fix 1.2 — Fix CORS for Published Domain

The backend must allow requests from BOTH the preview URL and the published URL:

```ts
// src/index.ts
import cors from 'cors';

const allowedOrigins = [
  'http://localhost:5173',
  'http://localhost:3000',
  process.env.FRONTEND_URL,           // published Replit URL
  /\.repl\.co$/,                       // all *.repl.co subdomains
  /\.replit\.dev$/,                    // Replit dev domains
  /\.kirk\.repl\.co$/,                 // Replit published variants
].filter(Boolean);

app.use(cors({
  origin: (origin, callback) => {
    // Allow requests with no origin (curl, Postman, server-to-server)
    if (!origin) return callback(null, true);

    const allowed = allowedOrigins.some(pattern => {
      if (typeof pattern === 'string') return origin === pattern;
      if (pattern instanceof RegExp) return pattern.test(origin);
      return false;
    });

    if (allowed) return callback(null, true);

    console.warn(`🚫 CORS blocked origin: ${origin}`);
    callback(new Error(`CORS: origin ${origin} not allowed`));
  },
  credentials: true,
  methods: ['GET', 'POST', 'PUT', 'PATCH', 'DELETE', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization', 'X-Requested-With'],
  exposedHeaders: ['Set-Cookie'],
}));

// Handle preflight for all routes
app.options('*', cors());
```

If using JWT in Authorization header (recommended for Replit):


```ts
// Frontend: store in localStorage, send as Bearer token
localStorage.setItem('slh_token', token);

// In API client — attach to every request
const apiCall = async (endpoint: string, options: RequestInit = {}) => {
  const token = localStorage.getItem('slh_token');
  const res = await fetch(`${API_BASE}${endpoint}`, {
    ...options,
    headers: {
      'Content-Type': 'application/json',
      ...(token ? { Authorization: `Bearer ${token}` } : {}),
      ...options.headers,
    },
  });

  if (res.status === 401) {
    localStorage.removeItem('slh_token');
    window.location.href = '/auth';
    return;
  }

  return res;
};
```

If using cookies instead, ensure these flags for HTTPS:
```ts
res.cookie('token', jwtToken, {
  httpOnly: true,
  secure: process.env.NODE_ENV === 'production', // HTTPS only in prod
  sameSite: process.env.NODE_ENV === 'production' ? 'none' : 'lax', // 'none' required for cross-origin
  maxAge: 7 * 24 * 60 * 60 * 1000, // 7 days
});
```

### Fix 1.6 — Auth Middleware Must Work With Bearer Token

```ts
// src/middleware/requireAuth.ts
import jwt from 'jsonwebtoken';
import type { Request, Response, NextFunction } from 'express';

export interface AuthRequest extends Request {
  user?: { userId: string; role: string; };
}

export const requireAuth = (req: AuthRequest, res: Response, next: NextFunction) => {
  // Check Authorization header first
  const authHeader = req.headers.authorization;
  let token = authHeader?.startsWith('Bearer ') ? authHeader.slice(7) : null;

  // Fallback: check cookie
  if (!token && req.cookies?.token) {
    token = req.cookies.token;
  }

  if (!token) {
    return res.status(401).json({ error: 'Authentication required' });
  }

  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET!) as { userId: string; role: string };
    req.user = decoded;
    next();
  } catch (err) {
    return res.status(401).json({ error: 'Invalid or expired token' });
  }
};
```

## 🔴 PART 2 — Fix Email Delivery (Invites & Access Approvals)

### Root Cause Analysis

Emails are not arriving because one or more of:
1. SMTP credentials are not in Replit Secrets (only in `.env`)
2. Gmail requires an **App Password**, not the regular account password
3. The SMTP connection is failing silently — errors are swallowed
4. Port 587 may be blocked on Replit — need to try port 465 with SSL
5. The `from` email address doesn't match the authenticated SMTP user
6. Nodemailer transport is created but never verified

---

### Fix 2.1 — Create a Robust Email Service

Replace whatever email code exists with this complete implementation:

```ts
// src/services/email.ts
import nodemailer from 'nodemailer';

// Create transporter — try multiple configurations
const createTransporter = () => {
  const host = process.env.SMTP_HOST || 'smtp.gmail.com';
  const port = parseInt(process.env.SMTP_PORT || '587');
  const user = process.env.SMTP_USER;
  const pass = process.env.SMTP_PASS;

  if (!user || !pass) {
    console.error('❌ Email: SMTP_USER or SMTP_PASS not configured in Secrets');
    return null;
  }

  return nodemailer.createTransport({
    host,
    port,
    secure: port === 465,     // true for 465 (SSL), false for 587 (STARTTLS)
    requireTLS: port === 587,
    auth: { user, pass },
    tls: {
      rejectUnauthorized: false, // allow self-signed certs (needed on some Replit envs)
    },
    connectionTimeout: 10000,
    greetingTimeout: 5000,
    socketTimeout: 10000,
  });
};

let transporter = createTransporter();

// Verify connection on startup
export const verifyEmailConnection = async (): Promise<boolean> => {
  if (!transporter) {
    console.error('❌ Email transporter not created — check SMTP_USER and SMTP_PASS in Secrets');
    return false;
  }
  try {
    await transporter.verify();
    console.log('✅ Email service connected to', process.env.SMTP_HOST);
    return true;
  } catch (error: any) {
    console.error('❌ Email service connection failed:', error.message);
    console.error('   Host:', process.env.SMTP_HOST);
    console.error('   Port:', process.env.SMTP_PORT);
    console.error('   User:', process.env.SMTP_USER);
    console.error('   Hint: If using Gmail, generate an App Password at:');
    console.error('         https://myaccount.google.com/apppasswords');
    console.error('   Hint: Enable 2FA on your Google account first');
    console.error('   Hint: Try SMTP_PORT=465 if 587 fails');
    return false;
  }
};

// Generic send function with retry
const sendEmail = async (options: {
  to: string;
  subject: string;
  html: string;
  text?: string;
}): Promise<boolean> => {
  if (!transporter) {
    console.error('❌ Cannot send email — transporter not initialized');
    return false;
  }

  const fromName = 'Spark Lead Hub';
  const fromEmail = process.env.SMTP_USER!;

  try {
    const info = await transporter.sendMail({
      from: `"${fromName}" <${fromEmail}>`,
      to: options.to,
      subject: options.subject,
      html: options.html,
      text: options.text || options.html.replace(/<[^>]+>/g, ''),
    });
    console.log(`✅ Email sent to ${options.to} | MessageID: ${info.messageId}`);
    return true;
  } catch (error: any) {
    console.error(`❌ Email failed to ${options.to}:`, error.message);
    // Don't throw — email failure must never crash user operations
    return false;
  }
};
```
