# UI/UX Enhancement – Modernize Document Icons & Actions in Lead Documents Module

## Objective

The icons currently used in the **Lead Documents** section are very basic, outdated, and do not align with the premium UI/UX of the rest of the SparkLead application.

Replace the existing document and action icons with a modern, consistent icon set that matches the design language used throughout the system.

**Important:** This is purely a UI/UX enhancement. Do **not** modify any existing functionality, business logic, permissions, APIs, or document management workflows.

---

# Current Issues

The current document list uses simplistic icons that make the module feel unfinished.

Examples include:

- Generic file icons
- Weak upload icon
- Small external/open icon
- Basic delete icon
- No visual differentiation between file types

Compared to the rest of the system, this section feels visually inconsistent.

---

# Expected Improvements

Replace all document-related icons with modern, professional icons from the existing application icon library (preferably Lucide, Heroicons, or the project's current icon set).

The icons should:

- Match the overall design language
- Be crisp and modern
- Be consistent across Light and Dark themes
- Maintain equal sizing and spacing
- Have proper hover animations
- Include tooltips where appropriate

---

Instead of using one generic file icon, display icons based on the uploaded file type.

Examples:

### PDF
- FileText / FileTypePdf icon

### Word Documents
- FileText icon

- FileSpreadsheet icon

### Images
- Image icon

### PowerPoint
- Presentation icon

- Archive icon

### Unknown Files
- Generic File icon
