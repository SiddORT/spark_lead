# 🔐 ROLE-BASED VISIBILITY WITH ADMIN ASSIGNMENT SUPPORT

---

# 🎯 OBJECTIVE

Implement role-based lead visibility while ensuring:

👉 Admin users can STILL be assigned as:

* Lead Owner
* Lead Handler

👉 Admin users:

* Can view ALL leads
* Can ALSO appear in lead assignments

---

# 🧠 CORE PRINCIPLE

---

## 🚨 IMPORTANT

**Role (access) ≠ Assignment (ownership)**

| Concept                        | Purpose                 |
| ------------------------------ | ----------------------- |
| Role (Admin / Handler / Owner) | Controls data access    |
| Assignment (Owner / Handler)   | Controls responsibility |

---

# ✅ FINAL BEHAVIOUR

---

## 👑 ADMIN

* Can see ALL leads
* Can be assigned as:

  * Owner
  * Handler
* Appears in dropdowns
* Not restricted by filtering

---

## 👤 LEAD OWNER

* Can see ONLY leads where:

```js
ownerId === currentUser.id
```

---

## 👤 HANDLER

* Can see ONLY leads where:

```js
handlerId === currentUser.id
```

---

## 👤 USER WITH BOTH (Owner + Handler)

```js
{
  $or: [
    { ownerId: user.id },
    { handlerId: user.id }
  ]
}
```

---

# 🔧 BACKEND IMPLEMENTATION

---

## ✅ ACCESS CONTROL LOGIC

```js
const buildAccessQuery = (user) => {
  // Admin → full access
  if (user.role === "ADMIN") {
    return {};
  }

  // Owner + Handler
  if (user.role === "OWNER_HANDLER") {
    return {
      $or: [
        { ownerId: user.id },
        { handlerId: user.id }
      ]
    };
  }

  // Only Owner
  if (user.role === "LEAD_OWNER") {
    return { ownerId: user.id };
  }

  // Only Handler
  if (user.role === "HANDLER") {
    return { handlerId: user.id };
  }

  return {};
};
```
