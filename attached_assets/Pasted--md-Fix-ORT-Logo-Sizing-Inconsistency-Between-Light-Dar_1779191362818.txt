# Fix ORT Logo Sizing Inconsistency Between Light & Dark Themes

The ORT logo sizing is currently inconsistent across themes:

- In LIGHT MODE the logo appears too small
- In DARK MODE the logo appears extremely oversized
- Branding alignment is inconsistent between login page and sidebar
- Logo scaling is not controlled centrally
- The visual hierarchy between `ort_` and `SparkLead` becomes unbalanced

This needs to be corrected with a unified responsive branding system.

---

# REQUIRED FIX

Create a centralized logo sizing system and ensure BOTH light and dark theme logos render at the EXACT SAME visual size.

The logo container dimensions must remain fixed regardless of:
- theme
- image asset
- resolution
- aspect ratio

---

# ROOT CAUSE

Currently:
- the dark logo asset has larger internal whitespace / dimensions
- the light logo asset has tighter bounds
- the image is rendering based on intrinsic image dimensions
- there is no normalized container sizing

This causes:
- huge logo in dark mode
- tiny logo in light mode

---

# REQUIRED IMPLEMENTATION

## Create Fixed Logo Wrapper

ALL logos must render inside a controlled wrapper.

Example:

```tsx
<div className="brand-logo-wrapper">
  <img
    src={logoSrc}
    alt="ORT SparkLead"
    className="brand-logo"
  />
</div>
```

---

# REQUIRED SIDEBAR LOGO STYLING

## Sidebar Logo Wrapper

```css
.brand-logo-wrapper {
  display: flex;
  align-items: center;
  justify-content: flex-start;
  height: 42px;
  width: 220px;
  overflow: hidden;
}
```

---

## Sidebar Logo Image

```css
.brand-logo {
  max-height: 32px;
  width: auto;
  object-fit: contain;
  display: block;
}
```

IMPORTANT:
- NEVER use `height: 100%`
- NEVER use `transform: scale()`
- NEVER use theme-specific scaling
- NEVER allow intrinsic image dimensions to control UI size

---
