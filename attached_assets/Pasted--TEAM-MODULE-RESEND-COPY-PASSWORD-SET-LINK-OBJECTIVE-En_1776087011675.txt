# 🔐 TEAM MODULE – RESEND + COPY PASSWORD SET LINK

---

# 🎯 OBJECTIVE

Enhance the existing **Resend Password Link** feature by allowing Admin to:

1. 🔁 Send reset link via email
2. 📋 Copy reset link manually (for WhatsApp / direct sharing)

# 🔧 BACKEND CHANGES

---

## ✅ NEW API (IMPORTANT)

```http
POST /api/users/:userId/generate-password-link
```

---

## ✅ RESPONSE

```json
{
  "resetLink": "https://app.yourdomain.com/set-password/<token>"
}
```

---

## ✅ BACKEND LOGIC

Same as resend API BUT:

👉 Do NOT send email
👉 Only generate and return link

---

### IMPLEMENTATION

```js
const token = crypto.randomBytes(32).toString("hex");

const hashedToken = crypto
  .createHash("sha256")
  .update(token)
  .digest("hex");

user.passwordResetToken = hashedToken;
user.passwordResetExpires = Date.now() + 1000 * 60 * 30;

await user.save();

const resetLink = `${FRONTEND_URL}/set-password/${token}`;

return res.json({ resetLink });
```

# 🔧 FRONTEND IMPLEMENTATION

---

## ✅ COPY LINK FUNCTION

```js
const copyPasswordLink = async (userId) => {
  try {
    const res = await api.post(`/users/${userId}/generate-password-link`);
    
    await navigator.clipboard.writeText(res.data.resetLink);

    showToast("Password setup link copied");
  } catch (err) {
    showToast("Failed to generate link");
  }
};
```

---

## ✅ BUTTON


# 🔒 SECURITY RULES (VERY IMPORTANT)

---

## ⚠️ TOKEN INVALIDATION RULE

Every time link is generated:

👉 Old token must become invalid

✔ Already handled by overwriting token in DB

---

## ⚠️ EXPIRY

* 30 minutes max
* Configurable

---

## ⚠️ SINGLE USE

* After password set → clear token

---
