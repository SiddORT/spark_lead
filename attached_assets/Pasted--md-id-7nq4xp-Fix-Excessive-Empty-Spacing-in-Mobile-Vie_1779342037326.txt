# Fix Excessive Empty Spacing in Mobile View & Improve Profile Page Mobile Responsiveness

Several pages in the system still have major mobile responsiveness and spacing issues.

Screenshots have been reviewed for reference and understanding.

The current mobile layout contains:
- excessive unused whitespace
- large vertical gaps
- poorly optimized table positioning
- broken content hierarchy
- profile page layout issues

This makes the mobile experience feel unfinished and inconsistent with a professional SaaS application.

---

# CURRENT ISSUES

# 1. Huge Empty Gap Between Search Bar & Data Table

On multiple pages in mobile view:
- there is a very large empty area between:
  - search/filter controls
  - and the actual table/list data

This issue is visible in:
- Team Management
- Services
- Companies
- other table-based pages

---

# CURRENT BEHAVIOUR

Current layout structure appears like:

```txt
Search Bar

[ HUGE EMPTY SPACE ]

Data Table
```

This creates:
- broken visual flow
- wasted screen space
- poor mobile UX
- unnecessary scrolling

---

# EXPECTED BEHAVIOUR

The data table/list should appear immediately below:
- search bar
- filters
- tabs
- controls

with only proper spacing.

Expected spacing:

```txt
12px–20px maximum
```

NOT massive blank areas.

---

# REQUIRED FIX

Investigate:
- fixed heights
- min-height usage
- flex-grow issues
- empty placeholder containers
- incorrect responsive spacing
- mobile breakpoint CSS
- hidden desktop elements still reserving height

---

# IMPORTANT

This issue appears to happen because:
- desktop spacing/layout logic is still being applied on mobile
OR
- containers are reserving unnecessary height

---

# REQUIRED IMPLEMENTATION

# REMOVE ALL UNNECESSARY EMPTY MOBILE SPACE

Audit:
- wrappers
- cards
- table containers
- flex layouts
- grid layouts
- content placeholders
- loading skeleton containers

---

# CHECK FOR ISSUES LIKE

```css
min-height: 400px;
height: 100%;
flex: 1;
```

that are incorrectly applied to mobile layouts.

---

# REQUIRED MOBILE TABLE LAYOUT

Mobile table/list sections should follow:

```txt
Header
↓
Search/Filters
↓
Table/List Immediately Below
```

without large gaps.

---

# REQUIRED RESPONSIVE TABLE IMPROVEMENTS

On mobile:
- reduce excessive container padding
- remove desktop-only spacing
- optimize filter stacking
- improve table/list positioning
