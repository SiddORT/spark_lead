# 🔍 GLOBAL SEARCH ENHANCEMENT – ADD SERVICE NAME SUPPORT

---

# 🎯 OBJECTIVE

Enhance the existing **global search bar** to allow users to search leads using:

* Lead Name ✅
* Company Name ✅
* Handler Name ✅
* Owner Name ✅
* **Service Name** 🆕

👉 Service is already visible in the table, so search must include it.

---

# 🧠 EXPECTED BEHAVIOUR

---

## ✅ SEARCH SHOULD MATCH AGAINST:

```js
[
  lead.leadName,
  lead.companyName,
  lead.handlerName,
  lead.ownerName,
  lead.serviceName   // NEW
]
```

---

## ✅ SEARCH FEATURES

* Case insensitive
* Partial match supported
  ("crm" → CRM Implementation)
* Works in real-time
* Ignores null/undefined values

---

# 🔧 BACKEND / DATA REQUIREMENTS

---

## 1. ENSURE SERVICE NAME EXISTS

Each lead object must include:

```json
{
  "leadName": "RVNL Dashboard Project",
  "companyName": "OneRoof Technologies",
  "handlerName": "Chirag",
  "ownerName": "Admin",
  "serviceName": "Software Application"
}
```

---

## ⚠️ IMPORTANT CASE

If service is stored as ID:

```json
"serviceId": "123"
```

👉 Then map before filtering:

```js
lead.serviceName = servicesMap[lead.serviceId]?.name;
```

---

# 🔧 SEARCH FUNCTION (FINAL)

---

```js
const searchText = input.toLowerCase().trim();

const filteredLeads = leads.filter((lead) => {
  return [
    lead.leadName,
    lead.companyName,
    lead.handlerName,
    lead.ownerName,
    lead.serviceName   // added
  ]
    .filter(Boolean)
    .some(field =>
      field.toLowerCase().includes(searchText)
    );
});
```
