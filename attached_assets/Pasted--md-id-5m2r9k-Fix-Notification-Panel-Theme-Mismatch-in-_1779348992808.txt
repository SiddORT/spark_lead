# Fix Notification Panel Theme Mismatch in Light Mode

The notification panel is currently NOT respecting the selected application theme.

When the system is in LIGHT MODE:
- the dashboard
- cards
- layout
- background
- components

all correctly switch to light theme,

BUT:

the notification panel still opens in DARK MODE.

This creates:
- inconsistent UI
- broken theme synchronization
- visually disconnected experience
- poor enterprise SaaS polish

Screenshots have been reviewed for reference and understanding.

---

# CURRENT ISSUE

# LIGHT MODE

Main application:
✅ Light theme

Notification panel:
❌ Dark theme

---

# DARK MODE

Main application:
✅ Dark theme

Notification panel:
✅ Dark theme

So the issue specifically affects:
- light theme synchronization
- notification popup styling
- theme propagation

---

# EXPECTED BEHAVIOUR

The notification panel MUST dynamically follow the currently selected theme.

---

# REQUIRED THEME BEHAVIOUR

# LIGHT MODE

Notification panel should use:

## Background

```txt
White / light neutral background
```

NOT:
```txt
Black / dark blue background
```

---

## Text

Use:
- dark readable text
- muted secondary text
- light borders

---

## Cards / Rows

Use:
- light hover states
- soft borders
- subtle shadows

---

## Badges

Maintain:
- proper contrast
- theme consistency

---

## Buttons

Use:
- same light mode button styling as rest of app

---

# DARK MODE

Keep existing dark notification panel design.

---

# ROOT CAUSE TO INVESTIGATE

The notification panel is likely:
- hardcoded to dark theme styles
OR
- not consuming global theme context/state
OR
- using fixed Tailwind classes

Examples of likely problems:

```tsx
bg-black
bg-[#0B1120]
text-white
```

instead of:

```tsx
theme-aware classes
```

---

# REQUIRED FIX

# MAKE NOTIFICATION PANEL FULLY THEME-AWARE

Notification system must dynamically respond to:
- global theme state
- theme provider
- theme context
- CSS variables
- dark/light class toggles

depending on current architecture.
