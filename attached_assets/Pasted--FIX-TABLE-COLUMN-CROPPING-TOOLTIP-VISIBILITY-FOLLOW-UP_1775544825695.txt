# ✅ FIX 1: PREVENT TABLE CROPPING

---

```tsx

```

---

## 🔹 STEP 2: DEFINE COLUMN WIDTHS

```tsx
<th className="w-[18%]">Lead Name</th>
<th className="w-[10%]">Type</th>
<th className="w-[16%]">Service</th>
<th className="w-[16%]">Company</th>

<th className="w-[12%]">Handler</th>

<th className="w-[12%] text-right">Follow-up</th>
```

---

## 🔹 STEP 3: ALIGN LAST COLUMN

```tsx
<td className="text-right whitespace-nowrap">
  {lead.followUpDate}
</td>
```

---

## 🔹 STEP 4: ENSURE TABLE DOES NOT OVERFLOW

```tsx
<div className="w-full rounded-xl border border-gray-800 overflow-visible">
```

---

## ❗ IMPORTANT

DO NOT USE:

```tsx

```

This is causing column clipping.

# ✅ FIX 2: TOOLTIP NOT VISIBLE (+1 COMPANY)

---

## 🚨 ROOT CAUSE

Tooltip is getting clipped due to:

* Parent `overflow-hidden`
* Low `z-index`
* Incorrect positioning

---

## 🔹 STEP 1: FIX PARENT CONTAINER

Ensure NO clipping:

```tsx
<div className="overflow-visible">
```

---

## 🔹 STEP 2: TOOLTIP WRAPPER

```tsx
<div className="relative group">
```

---

## 🔹 STEP 3: TOOLTIP UI

```tsx
<div className="absolute z-50 hidden group-hover:block 
                bg-gray-900 border border-gray-700 
                text-sm text-white px-3 py-2 rounded-lg 
                shadow-xl whitespace-nowrap
                top-full mt-2 left-0">
  {additionalCompanies.join(", ")}
</div>
```

---

## 🔹 STEP 4: ENSURE NO PARENT BREAKS TOOLTIP

Check ALL parents:

❌ Remove if present:

```tsx
overflow-hidden
overflow-x-hidden
```

---

## 🔹 STEP 5: OPTIONAL (BEST UX)

